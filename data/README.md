# Data Folder

## raw/
Current scope is one school, one term: `excella_school_a_term1.csv`
(Excella Secondary School Rwanda, anonymized to "School A"). There is no
School B and no Term 2/3 data yet.

If a new school or term is ever collected, drop it here matching the pattern
`*_term*.csv` — `data/preprocess.py` globs this pattern and concatenates
every file it finds. Columns must already match the Go Academics schema (see
processed/ below) — the real school name is the only thing preprocess.py
remaps (via `SCHOOL_CODES`).

The UCI Student Performance Dataset (`student-mat.csv`, `student-por.csv`) is
kept here as Plan B — see docs/plan_b.md. Don't mix it into real school term
files. It's no longer purely a standalone fallback: `data/preprocess_merge.py`
merges it with the Excella data for **training only** (see below).

## processed/
Cleaned, anonymized data ready for the ML pipeline. Matches this column
structure (see CLAUDE.md) — Subject/Stream values vary by school and are not
fixed to a hardcoded list:

```
StudentID | School | Gender | Stream | Term | Subject | CA_Score | Exam_Score | Attendance_Pct | Final_Result
```

- `students.csv` — **Excella only**, written by `preprocess.py`. This is the
  file `data/seed.py` reads to populate the live demo database — it must only
  ever contain real, anonymized Excella students, never synthetic UCI rows.
- `students_combined.csv` — Excella + UCI merged, human-readable, with an
  extra `Source` column ("Excella"/"UCI") for traceability. Written by
  `preprocess_merge.py`. Not used by `seed.py`.
- `students_ml.csv` — encoded + SMOTE-balanced, training-ready. After running
  `preprocess_merge.py` this is the **combined** dataset (overwrites the
  Excella-only version `preprocess.py` alone would produce).
- `encodings.json` — LabelEncoder class mappings, regenerated by whichever
  script ran last (`preprocess.py` = Excella-only, `preprocess_merge.py` =
  combined). `train.py` embeds whichever is current into `model_meta.json` so
  inference never has to hardcode category values.

**Regenerating the combined training set:** run in this order —
`python data/preprocess.py` (refreshes `students.csv` for seeding) →
`python data/preprocess_merge.py` (rebuilds `students_ml.csv`/`encodings.json`
from Excella + UCI) → `python backend/app/ml/train.py`. Running
`preprocess_plan_b.py` standalone after `preprocess.py` would overwrite
`students.csv` with UCI-only data — don't do that; `preprocess_merge.py`
imports both scripts' logic in-memory specifically to avoid this.

Never commit raw files containing real student names — only School A /
School B anonymized data belongs in version control.
